# 命理溯源 — UI 重设计规格文档

**日期**：2026-06-05
**状态**：设计完成，待实现

---

## 1. 概述

### 1.1 目标

将"命理溯源"前端界面从基础中式风格升级为**专业级中式美学界面**，达到全历史 App、潮汐 App 级别的设计质量，为视频推广和公开部署做准备。

### 1.2 设计参考

- **全历史 App**：首页内容卡片流 + 宫格入口 + 时间轴交互
- **潮汐 App**：极简中式留白 + 禅意动效 + "当下"页面设计
- **问真八字 App**：工具页极简专注 + 三面板展示 + 输入→结果直达
- **故宫博物院官网**：配色与纹样系统（长期参考）

### 1.3 当前状态

- 功能层面：排盘、宜忌、风水、周易、典籍、学堂等核心模块均已实现
- 设计层面：有基础中式配色（宣纸白/朱砂红/鎏金/墨色），但细节粗糙
- 导航层面：5 Tab（命理/择吉/风水/周易/典籍），信息架构不够清晰

## 2. 设计系统

### 2.1 设计方向：宣纸书院

白底 + 暖色纸纹 + 极简中式留白。像翻开一本古籍善本，明亮清雅，白天使用舒适。

### 2.2 配色方案

| Token | 色值 | 用途 |
|-------|------|------|
| 宣纸白 | `#F5F0E8` | 页面背景 |
| 卡片白 | `#FFFFFF` | 卡片、输入区背景 |
| 朱砂红 | `#C43A31` | 标题、重要标识、主按钮、日柱高亮 |
| 鎏金 | `#C9A96E` | 边框、分隔线、高级标识、当前章节 |
| 墨色 | `#2C2416` | 正文文字 |
| 旧纸灰 | `#8B7D6B` | 次要说明、日期信息 |
| 靛蓝 | `#3D5A80` | 链接、可点击元素（保留现有） |
| 浅宣纸 | `#EDE5D5` | 卡片内分隔、锁定状态背景 |
| 浅边框 | `#E0D5C0` | 卡片边框 |

### 2.3 字体

- 标题/展示：ZCOOL_XiaoWei（站酷小薇体，现有）
- 正文：系统默认中文字体栈

### 2.4 动效系统：禅意节奏

- 页面切换：淡入淡出，600ms ease-out
- 卡片入场：从下浮出（fade-up），元素间交错延迟 80ms
- 数字变化：慢速计数动画
- 按钮反馈：微缩放 + 颜色过渡
- 整体气质：慢、柔、有呼吸感，给用户"不赶时间"的感觉

## 3. 信息架构

### 3.1 导航：3 Tab

```
┌──────────┬──────────┬──────────┐
│  🏮 今日  │  ☯ 排盘  │  📚 学堂  │
└──────────┴──────────┴──────────┘
```

| Tab | 定位 | 核心内容 |
|-----|------|---------|
| 今日 | 内容首页 | 日历 Hero + 宜忌卡片 + 宫格入口 + 每日古籍 + 历史命例 |
| 排盘 | 核心工具 | 八字排盘 + 格局 + 用神 + AI 古籍解读 |
| 学堂 | 学习系统 | 进度追踪 + 六章体系课程 + 测验 + 场景跳转入口 |

### 3.2 被合并的页面

原 5 Tab 中的**择吉、风水、周易、典籍**功能不再作为独立 Tab，而是：
- 择吉 → 合并到"今日"首页（黄历宜忌本身就是择吉的结果）
- 风水/周易/典籍 → 放在今日首页的宫格入口中，进入后为独立沉浸式工具页

**交互**：
- 输入区：支持真太阳时校正（城市输入），性别切换
- 结果区：Tab 切换（概览/详情/AI解读），保留现有实现
- 日柱高亮（朱砂红 + 放大）是视觉焦点
- 格局/用神以 chip 形式展示，可点击弹出详解
- "想深入了解"链接 → 跳转到学堂对应章节
- 分享卡片：生成四柱八字图片，可保存分享

### 4.3 学堂（学习系统）

**双轨设计**：
- **体系课程**：从学堂 Tab 进入 → 完整的六章学习路径
- **场景入口**：从排盘/今日/古籍引用处跳转 → 精准定位到相关章节

**布局**：

```
┌──────────────────────────┐
│  ┌────────────────────┐  │
│  │  ◉ 45%            │  │  ← 环形进度条 + 学习进度
│  │  已完成第2章        │  │
│  │  [继续学习 → 第3章] │  │
│  └────────────────────┘  │
│                          │
│  最近学习                │
│  📖 第2章 · 十神与格局    │
│  3天前 · 测验 8/10       │
│                          │
│  全部章节                │
│  ┌────────────────────┐  │
│  │ ☰ 第1章 阴阳五行 ✅  │  │
│  │ ☲ 第2章 十神格局 ✅  │  │
│  │ ☵ 第3章 用神调候 ▶  │  │  ← 鎏金边框高亮（当前）
│  │ ☳ 第4章 神煞辨识 🔒 │  │  ← 锁定状态
│  │ ☴ 第5章 大运流年 🔒 │  │
│  │ ☶ 第6章 古籍导读 🔒 │  │
│  └────────────────────┘  │
└──────────────────────────┘
```

**课程体系**（已有 data/learning_chapters.json）：
1. 阴阳五行基础
2. 十神与格局
3. 用神与调候
4. 神煞辨识
5. 大运流年
6. 古籍导读

**解锁机制**：
- 每章末尾有测验（题库已在 data/quiz_cases.jsonl）
- 通关分数 ≥7/10 解锁下一章
- 开发者模式：点标题 3 次全解锁（现有功能保留）

**场景跳转**：
- 排盘结果 → "七杀格是什么？" → 学堂第2章第3节
- 今日古籍卡片 → 学堂对应章节
- AI 解读底部引用 → 精准跳转古籍原文所在章节

**交互**：
- 章节卡片点击 → 进入该章教材内容（课文 + 测验）
- 已完成章节可随时回看
- 测验答题后 AI 导师解释（苏格拉底式引导，现有功能保留）

## 6. 数据流

```
今日页面：
  /api/yiji → CalendarHero（日期+干支+宜忌）
  /api/search → DailyAncientCard（每日古籍）
  /api/mingli → MingliStoryCard（随机命例）

排盘页面：
  /api/full-analysis → PaipanInput → 全量分析
  /api/sources → AncientRefCard（古籍来源）

学堂页面：
  /api/xuetang → ProgressRing + ChapterList
  /api/xuetang/:id → ChapterContent
  /api/quiz → QuizCard
  /api/mentor → AIMentor

场景跳转：
  ?from=paipan&ref=七杀格&chapter=2&section=3
  → 学堂第2章第3节高亮
```

## 8. 约束与不变量

- 技术栈不变：Next.js 16 + Tailwind CSS v4 + Framer Motion + TypeScript
- 现有 API 接口不变（前端重构只改 UI 层）
- CLAUDE.md 中定义的 AI 使用红线不变
- 配色 token 在现有基础上扩展，不推翻重来
- 移动端优先：基准 375px，最大宽度 430px
- 所有页面底部保留免责声明
